합병 정렬Merge Sort



합병 정렬(Merge Sort)은 입력이 2개의 부분문제로 분할되고, 부분문제의 크기가 1/2로 감소하는 분할 정복 알고리즘이다. 즉, n개의 숫자들을 n/2개씩 2개의 부분문제로 분할하고, 각각의 부분문제를 재귀적으로 합병 정렬한 후, 2개의 정렬한 부분을 합병하여 정렬(정복)한다.
즉, 합병 과정이 문제를 정복하는 것이다.

의사 코드

MergeSort(A,p,q)
입력: A[p]~A[q]
출력: 정렬된 A[p]~A[q]
  1. if ( p < q ) {
  2.  k = |(p+q)/2|
  3.  MergeSort(A,p,k) // 앞부분 재귀 호출
  4.  MergeSort(A,k+1,q) // 뒷부분 재귀 호출
  5.  A[p]~A[k]와 A[k+1]~A[q]를 합병한다.
  6. }

시간 복잡도

분할하는 부분은 배열의 중간 인덱스 계산과 2번의 재귀 호출을 하므로 O(1)이다.
합병을 하는 수행 시간은 입력의 크기에 비례한다. 즉, 2개의 정렬된 배열 A와 B의 크기가 각각 n과 m이라면, 최대 비교 횟수는(n+m-1)이다. 왜냐하면 합병하는 데 2개의 숫자를 1번 비교할 때마다, 하나의 승자가 탄생하고, 승자는 합병된 배열 C에 저장되기 때문에 따라서 배열 C에는 결국 배열 A와 B의 모든 숫자들이 저장되나, 가장 마지막에 저장되는 숫자는 비교할 숫자가 없으므로 합병의 시간복잡도는 O(m+n) 이다.
그러나 n을 계속하여 2로 나누다가 더이상 나눌 수 없는 크기인 1이 될 때 분할을 중단하는데 따라서 logn의 층 수가 생긴다.
따라서 합병 정렬의 시간 복잡도는 O(nlogn)이다.

활용

합병 정렬은 외부정렬의 기본이 되는 정렬 알고리즘으로 연결 리스트에 있는 데이터를 정렬할 때에도 퀵 정렬이나 힙 정렬보다 훨씬 효율적이다. 또한 멀티코어 CPU와 다수의 프로세서로 구성된 그래픽 처리 장치의 등장으로 정렬 알고리즘을 병렬화하는 데에 합병 정렬 알고리즘이 활용된다.
