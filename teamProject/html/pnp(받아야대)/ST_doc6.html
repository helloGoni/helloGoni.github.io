힙 정렬HeapSort



힙(Heap)은 힙 조건을 만족하는 완전 이진트리이다. 힙 조건이란 각 노드의 값이 자신 노드의 값보다 커야한다. 노드의 값은 우선순위(priority)라고 일컫는다. 따라서 힙의 루트에는 가장 높은 우선순위가 저장되어있다.

힙 정렬(Heap Sort)은 앞에서 설명한 힙 자료구조를 이용하는 정렬 알고리즘이다. 오름차순 정렬을 위해 입력 배열을 먼저 큰 숫자가 높은 우선 순위를 가지는 최대힙을 만든다. 힙의 루트에는 가장 큰 수가 저장되므로, 루트의 숫자를 힙의 가장 마지막 노드에 있는 숫자와 바꾼다. 그리고 힙 크기를 1개 줄인 다음에, 루트에 저장된 숫자로 인해 위배된 힙 조건을 해결하여 힙 조건을 만족시킨다. 그리고 이 과정을 반복하여 나머지 숫자를 정렬한다.

의사 코드

입력: 크기가 n인 배열
출력: 정렬된 배열 A
  1. 배열 A의 숫자에 대해서 힙 자료구조를 만든다.
  2. heapSize = n
  3. for i = 1 to n-1
  4.  swap(A[1], A[heapSize]
  5.  heapSize--
  6.  DownHeap() // 위배된 힙 조건을 만족시킨다.
  7. return 배열A

시간 복잡도

Line 1에서 힙을 만드는데는 O(n)시간이 걸린다. Line 2는 변수초기화이므로 O(1)시간이 걸린다. Line 3~6의 for 루프는 n-1번 수행되고, 루프 내부에서는 line 4~5가 각각 O(1)시간이 걸린다. DownHeap은 최악의 경우 이파리노드까지 내려가며 교환할 때이고, 힙의 높이는 log2n을 넘지 않으므로, O(log2n)시간이 걸린다. 따라서 힙 정렬의 시간 복잡도는 O(n) + O(n - 1) × O(log2n) = (nlog2n)
