작업 스케줄링Task Scheduling

작업 스케줄링(Task Scheduling) 문제는 작업의 수행 시간이 중복되지 않도록 모든 작업을 가장 적은 수의 기계에 배정하는 문제이다.
최적해를 찾기 위해서는 빠른 시작시간 작업을 우선 배정 하는 방법을 사용해야 한다.

작업 스케줄링은 비즈니스 프로세싱, 공장 생산 공정, 강의실/세미나룸 배정, 컴퓨터 태스크 스케줄링 등에도 운영된다.

의사 코드

입력: n개의 작업 t1, t2 ... tn
출력: 각 기계에 배정된 작업 순서
  1. 시작시간의 오름차순으로 정렬한 작업 리스트를 L이라고 하자.
  2. while(L != 공집합) {
  3.  L에서 가장 이른 시작시간을 가진 작업 t를 가져온다.
  4.  if (t를 수행할 기계가 있으면)
  5.   t를 수행할 수 있는 기계에 배정한다.
  6.  else
  7.   새로운 기계에 t를 배정한다.
  8.  t를 L에서 제거한다.
  9.  }
  10. return 각 기계에 배정된 작업 순서

시간 복잡도

Line 1에서 n개의 작업을 정렬할 때 O(nlogn) 시간이 걸리고, while-루프에서 작업을 L에서 가져다 수행가능한 기계를 찾아 배정하므로 O(m) 시간이 걸린다. 이 루프가 총 n번 시행되므로 line 2~9까지 O(mn) 시간이 걸린다.
따라서, TaskScheduling 알고리즘의 시간복잡도는 O(nlogn)+O(mn)이다.
