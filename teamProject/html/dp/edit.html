편집 거리 문제EditDistance

편집 거리(Edit Distance) 문제는 문서 편집기를 사용하는 중에 스트링 S를 T로 변환시키고자 할 때, 삽입(insert), 삭제(delete), 대체(substitute) 연산을 최소로 사용하는 횟수를 구하는 문제이다.

부분문제를 정의하면 E[i, j]는 S의의 접두부의 i개의 문자를 T의 접두부 j개 문자로 변환시키는데 필요한 편집 거리를 의미한다.

부분문제를 구하는 방법은, 예를 들어서 E[i,j]의 경우 E[i,j-1]에서 수정 연산을 한번 더 하거나, E[i-1,j]에서 수정 연산을 한번 더 하거나 E[i-1,j-1]이 E[i,j]이 될 때 수정 연산을 하는 경우 중에서 가장 작은 값을 찾는 것이다.

시간 복잡도

편집 거리 알고리즘의 시간 복잡도는 O(mn)이다. 여기서 m과 n은 두 스트링의 각각의 길이이다. 그 이유는 총 부분 문제의 수가 배열 E의 원소의 수인 m × n이고, 각 부분 문제를 계산하기 위해서 주변 3개의 부분문제들의 해를 참조한 후 최솟값을 찾는 것이므로 O(1) 시간이 걸리기 때문이다.

사용처

편집 거리 문제의 응용 사례는 매우 다양하다. 2개의 스트링들 사이의 편집 거리가 작으면 이 스트링들이 서로 유사하다고 판단할 수 있으므로, 생물 정보 공학(Bioinformatics) 및 의학 분야에서 두 개의 유전자가 유사한지 측정하는데 사용된다. 그 외에서 철자 오류 검색(Spell Checker), 자연어 번역(Natural Language Translation) 소프트웨어 등에도 활용된다.
