연속 행렬 곱셈

연속 행렬 곱셈(Chained Matrix Multiplication) 문제는 연속된 행렬에서의 곱셈에 필요한 원소 간의 최소 곱셈 수를 찾는 문제이다.

두 개의 행렬의 경우, 예를 들어 10 × 20의 행렬과 20 × 5의 행렬의 곱을 구하는 경우 필요한 곱셈의 횟수는 10 × 20 × 5 = 1000이다.

그러나 세 개의 경우, 예를 들어 10 × 20의 행렬과 20 × 5 행렬 그리고 5 × 15의 곱을 구하는 경우, A와 B를 먼저 곱하는지, 아니면 B와 C를 먼저 곱하는지에 따라서 수행해야하는 곱셈의 크기가 달라진다. 그러므로 곱셈을 최소화하기 위해서는 적절한 순서를 찾을 필요가 있다.

이 문제는 동적 계획 알고리즘을 이용하여 해결할 수 있다.
구해야하는 부분 문제는 예를 들어서 A × B × C × D × E를 구하는 경우, A × B, B × C, C × D, D × E를 구하고, 이를 이용하여 A × B × C, B × C × D, C × D × E를 구하고 이를 반복하여 최종적으로 A × B × C × D × E의 최솟값을 구할 수 있다.

의사 코드

입력: 연속된 행렬 A1 × A2 ... n 단, A1 = d0 × d1이고, An = dn-1 × dn이다.
출력: 입력의 행렬곱셈에 필요한 원소 간의 최소 곱셈 횟수
  1. for i = 1 to n
  2.  C[i,i] = 0
  3. for L = 1 to n - 1 {
  4.  for i = 1 to n-L {
  5.   j = i + L
  6.   C[i, j] = ∞
  7.   for k = i to j-1 {
  8.    temp = C[i, k] + C[k+1, j] + di-1dkdj {
  9.    if(temp < C[i,j])
  10.   C[i,j]=temp
  11.   for k = i to j-1 }}}
  12. return C[1, n]

시간 복잡도

MatrixChain 알고리즘의 시간 복잡도는 배열 C가 n × n이므로 거의 1/2 정도의 원소의 값을 계산하여야한다. 그런데 하나의 원소를 계산하기 위해서는 Line 7의 루프가 최대 n-1번 수행되어야하므로 O(n-1) × O(n2) -> O(n3)이다.
